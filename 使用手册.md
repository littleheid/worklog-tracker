# Worklog Tracker 使用手册 v0.2.0

## 安装

### macOS
```bash
brew install littleheid/worklog-tracker/worklog-tracker
```

### Windows
```powershell
scoop bucket add littleheid https://github.com/littleheid/scoop-worklog-tracker
scoop install worklog-tracker
```

### Linux
从 [GitHub Releases](https://github.com/littleheid/worklog-tracker/releases/latest) 下载对应架构二进制。

```bash
chmod +x worklog-tracker_*_linux_amd64
./worklog-tracker_*_linux_amd64 start
```

安装后自动注册为开机自启服务，每次登录自动运行。

---

## 基本命令

| 命令 | 说明 |
|------|------|
| `worklog-tracker start` | 启动服务 |
| `worklog-tracker stop` | 停止服务 |
| `worklog-tracker status` | 查看运行状态 |
| `worklog-tracker restart` | 重启服务 |
| `worklog-tracker install` | 注册开机自启 |
| `worklog-tracker version` | 查看版本号 |

## 访问

启动后打开浏览器访问 **http://localhost:4173**

## 功能说明

### 首页仪表盘
- **KPI 卡片**：总事项数、已完成数、今日到期、已逾期
- **事项卡片网格**：按状态分组（待办/进行中/已完成）
- 卡片 hover 显示操作按钮（编辑/删除/状态切换）
- 点击卡片查看详情面板

### 事项管理
- **新增事项**：点击右上角「新增」按钮
- **编辑事项**：卡片上 hover 后点编辑图标
- **删除事项**：卡片上 hover 后点删除图标，需确认
- **状态切换**：待办 → 进行中 → 已完成
- **批量导入**：支持一次性粘贴多条事项，每行一个标题

### 表单字段

| 字段 | 必填 | 说明 |
|------|------|------|
| 标题 | ✓ | 最多 200 字 |
| 内容说明 | × | 最多 5000 字 |
| 状态 | × | 待办 / 进行中 / 已完成 |
| 优先级 | × | 低 / 中 / 高 |
| 标签 | × | 自定义标签，支持输入多个 |
| 截止日期 | × | 可选，不设置则不参与逾期判断 |

### 标签系统
- 输入标签后按回车确认
- 每个标签自动分配 8 种颜色之一（根据标签名 hash）
- 标签使用空心样式，与状态/优先级徽章区分明显
- 支持删除已有标签（点击 ×）

### 截止日期颜色

| 状态 | 颜色 |
|------|------|
| 已逾期 | 🔴 红色 |
| 今天到期 | 🟠 琥珀色 |
| 未到期 / 无日期 | ⚪ 浅色 |

### 修复：页面切换卡取
- 页面切换使用 `fade` 过渡动画
- 布局切换不跳动

### 洞察页
- **仪表盘图表**：完成率仪表盘
- **环形图**：状态分布（待办/进行中/已完成）
- **柱状图**：按月统计事项数量

### 设置页
- 语言切换（中文 / English）
- 每页显示数量（10/20/50/100）

## 数据存储

- 数据库位置：`~/.worklog-tracker/worklog.db`（SQLite）
- 配置文件：`~/.worklog-tracker/prefs.json`
- 服务 PID：`~/.worklog-tracker/worklog.pid`
- launchd plist：`~/Library/LaunchAgents/com.worklog.tracker.plist`

### 备份数据

```bash
cp ~/.worklog-tracker/worklog.db ~/Desktop/worklog-backup.db
```

## 卸载

```bash
brew uninstall worklog-tracker
brew untap littleheid/worklog-tracker
```

数据目录保留在 `~/.worklog-tracker/`，需手动删除：

```bash
rm -rf ~/.worklog-tracker
```

## 故障排除

### 页面无法访问
```bash
worklog-tracker status    # 查看服务状态
worklog-tracker restart   # 尝试重启
```

### 新增失败
1. 按 `Cmd+Shift+R` 强制刷新浏览器
2. 确认标题字段已填写（必填）

### 数据库损坏
```bash
worklog-tracker stop
rm ~/.worklog-tracker/worklog.db
worklog-tracker start
```
重启后数据库自动重建（数据将丢失，请先备份）

### 查看日志

macOS 上通过 log 命令查看：

```bash
log show --predicate 'process == "worklog-tracker"' --last 10m
```
